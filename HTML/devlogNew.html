Devlog project 3DWorld

Voor dit project is de opdracht om een subscriptie gebaseerde website te maken,
waarop mensen 3Dprint-artiesten kunnen steunen DMV van een maandelijkse bijdrage.
In deze devlog documenteer ik mijn werk voor dit project.

Subscribe functie
(Datum van toepassing: 06-11-23)


Om dit project soepel te laten verlopen, is het natuurlijk de bedoeling dat ingelogde gebruikers kunnen abonneren op een artiest.
Mijn taak voor dit project was onder andere het maken van een subscribe functie die de subscriptie van de gebruiker vastlegt.

Deze webpagina representeert een artiest vanuit het perspectief van een ingelogde gebruiker.

-Als de ingelogde gebruiker zich abbonneert:

1. Wordt er een request naar de backend gestuurd die het desbetreffende request controlleert.

1A. Zodien het request valide is, wordt er een SQL querry gestuurd naar de database die de abonnee definitief vastlegt.

1B. Als het request niet valide is, wordt het request onderschept en krijgt de gebruiker een foutmelding te zien.

User flow

-Als de ingelogde gebruiker het abbonement opzegt:

1. Wordt er eerst gevraagd of de gebruiker de subscriptie wel degelijk wilt opzeggen DMV een prompt om te verzekeren dat het geen verkeerde input kan zijn vanuit de gebruiker.

1A. Zodien de gebruiker confirmeert dat diegene het abbonement op wilt zeggen wordt er een request naar de backend gestuurd die eerst controlleert of het een valide request is.
-Is het een valide request, dan wordt er een query verstuurd naar de database die het abonnement weghaalt.
-Is het een invalide request, dan wordt er geen query verstuurd naar de database.

1B. Als de gebruiker annuleert.
-wordt het proces onderschept en blijft degene geabonneert.
